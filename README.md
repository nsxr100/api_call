FIRST, CREATE A NEW LARAVEL PROJECT USING THE COMMAND:

LARAVEL NEW PROJECTNAME

AFTER CREATING THE PROJECT, GENERATE THE NECESSARY FILES USING ARTISAN COMMANDS SUCH AS THE MODEL, MIGRATION, AND CONTROLLER FOR THE STUDENTS MODULE. THEN CONFIGURE THE API ROUTES INSIDE THE API.PHP FILE TO HANDLE ALL STUDENT-RELATED REQUESTS.

NEXT, CREATE AND STRUCTURE THE STUDENTS TABLE USING LARAVEL MIGRATIONS, AND IF NEEDED, USE LARAVEL TINKER TO TEST OR MANAGE DATA DIRECTLY FROM THE COMMAND LINE.

ONCE THE BACKEND SETUP IS COMPLETE, OPEN POSTMAN AND CREATE A WORKSPACE AND COLLECTION TO ORGANIZE ALL API REQUESTS.

INSIDE THE COLLECTION, SET UP THE FOLLOWING REQUESTS:

GET – RETRIEVES AND DISPLAYS ALL STUDENT RECORDS IN JSON FORMAT.

POST – ADDS A NEW STUDENT BY SUBMITTING NAME, COURSE, AND EMAIL.

PUT – UPDATES THE COMPLETE INFORMATION OF A STUDENT USING THEIR ID.

PATCH – UPDATES ONLY SPECIFIC FIELDS OR SELECTED DATA OF A STUDENT.

DELETE – REMOVES A STUDENT RECORD BY ID OR DELETES ALL RECORDS IF REQUIRED.

RUN THE URL HTTP http://127.0.0.1:8000/api/students/
